| 1.    设计（或研究）的依据与意义  （1）研究依据  ①政策依据：当前国家政策持续聚焦数字经济、文化创新与个人能力发展，为课题研究提供了明确的政策导向。《“十四五” 数字经济发展规划》明确提出 “促进数字内容创作创新，支持个人创作者等群体发展数字内容生产”，鼓励搭建面向普通创作者的轻量化数字内容管理与分享载体；《关于推进实施国家文化数字化战略的意见》强调要 “构建便捷高效的个人数字文化内容存储、管理与交流渠道”，助力全民文化创新实践；同时，《知识产权强国建设纲要（2021 - 2035 年）》提出要 “完善个人创作成果保护与展示机制”，为本平台的作品管理、分享功能提供了政策合规性支撑。上述政策均指向对个人数字创作的支持与规范，为本课题的开展提供了坚实的政策基础。  ②技术层面依据：前后端分离的 B/S 架构已在 Web 应用开发领域趋于成熟并广泛应用，其中 Spring Boot 作为后端开发框架、Vue 作为前端开发框架的技术组合具备显著优势。该架构可通过 RESTful 接口实现系统解耦，不仅能稳定支撑用户管理、创作记录、作品展示与互动等核心功能模块的开发实现，还具备良好的可扩展性和工程可行性，有效降低了开发难度与后期维护成本，为平台落地提供了扎实的技术支撑。  ③现实需求依据：在知识经济与数字内容生产快速发展的背景下，学生、初级创作者等群体存在强烈的核心需求 —— 既需要集中存放个人作品、记录创作灵感、梳理创作思路，也希望获得适度的分享与交流空间。但当前主流内容平台以商业内容分发和流量推荐为核心，存在界面复杂、使用成本高、干扰信息多等问题，更侧重内容消费与成果曝光，难以满足个人创作的系统化管理、长期归档及纯粹交流需求，这都为课题研究提供了明确的方向。  （2）研究意义  ①理论意义  本课题将软件工程理论与个人创作管理的具体应用场景深度结合，聚焦内容管理系统在个人创作领域的实践应用。课题通过引入前后端分离架构与模块化设计思想，对用户管理、创作记录、作品展示与互动等功能进行分析与实现，丰富了内容管理系统在个人创作与数字内容管理方向的应用研究边界。  课题围绕 “创作过程记录” 核心需求展开系统设计，强调创作过程与创作成果的统一管理，为个人知识管理理论在实际系统中的落地提供了可参考的工程实践案例，同时，对同类系统的设计思路与实现具有一定的学术参考价值。  ②实践意义  针对当前主流平台商业属性强、干扰信息多、创作记录功能不足等问题，本课题设计并实现轻量化、高易用性的个人创作记录与作品分享平台，为学生和初级创作者提供了简单的创作管理工具。用户可通过平台进行系统化作品整理、记录创作过程与想法，同时实现分享与交流，有效提升创作管理效率与持续创作动力。  平台具备良好的可扩展性，后续可进一步拓展为小型创作社区或校园作品展示平台，这些契合了政策对全民数字创作的支持导向，具备广泛的推广与应用价值，能够为更多创作群体提供合规、便捷的服务。 |                                                              |          |                |          |          |
| 2.    国内外同类设计（或同类研究）的概况综述   **（1）**    **国内研究现状**  国内相关研究从个人知识管理方法、数字化创作产品体验以及工程实现路径等方面对个人内容记录与管理进行了有益探索，强调了记录的结构化、功能闭环设计以及系统可实现性的重要性。但现有研究多聚焦于单一工具或通用内容平台，对以个人创作过程记录为核心、兼顾作品管理与分享的轻量化平台研究仍相对不足，这也为本课题的研究与系统设计提供了明确的实践空间。  如在“个人知识管理工具与记录机制”方面，朱志赟、谢子晴（2024）在《知识管理论坛》发表的研究《基于Zotero的个人知识管理最佳实践》中，提出个人知识管理可按“获取—整理—吸收—应用”过程展开，并归纳了理想工具应具备的关键能力（如元数据规范、分类组织、可回溯与定期复盘等），强调“记录的结构化与可回顾性”是效率提升的重要来源。因此，个人创作记录与作品分享平台在功能设计上不应仅停留在作品发布与浏览层面，而应围绕创作过程构建“记录—整理—展示—互动反馈”的轻量闭环。通过将标签与分类、创作说明、时间线管理以及检索与统计等功能作为平台的基础能力，有助于保障个人作品和创作灵感在长期积累过程中的可管理性和可回顾性，从而更好地支持个人创作的持续发展。  在“数字化知识/创作一体化产品体验”方面，何安怡（2023）在汉斯出版社相关研究中从用户体验视角分析数字出版/研学类产品的功能构成，指出此类产品往往围绕“阅读—创作—笔记—个人知识管理—资料整理”等环节形成闭环，并以功能分层方式服务用户的内容沉淀与再利用。该研究对本课题的意义在于：你的平台同样需要形成“记录—整理—展示—互动反馈”的轻量闭环，让普通创作者以更低的使用成本完成内容沉淀与呈现，而不是仅提供“发布—浏览”的单一链路。  在“工程实现与技术路线”方面，国内大量系统实践采用 Spring Boot + Vue 的前后端分离架构来实现内容管理与交流分享类平台。以阿里云开发者社区（2024）对“基于SpringBoot+Vue交流和分享平台”的工程实践介绍为例，其总结的流程涵盖需求分析、架构设计、数据库建模与模块实现，体现了该技术路线在中小型内容平台开发中的可落地性。对本课题而言，这说明选用 Spring Boot + Vue 构建“用户管理、作品管理、互动与后台管理”等模块具备成熟的工程参考路径，有利于在毕业设计周期内完成可运行系统并形成规范文档。  **（2）**    **国外研究现状**  国外对个人知识管理与内容创作支持平台的研究起步较早，相关研究主要围绕知识结构化方式、用户创作行为特征以及数字工具对学习与创作过程的支持作用展开。部分研究提出以用户为中心的内容管理模型，强调通过记录、整理与分享等功能的协同设计，支持个人知识的持续积累与能力发展。同时，这类研究普遍关注系统的易用性和长期使用体验，认为通过平台化设计降低使用门槛，有助于增强用户参与度和内容沉淀效果。上述研究成果为个人内容管理系统的整体架构和功能设计提供了较为成熟的理论与实践思路。  例如在内容平台机制与创作者行为关系研究方面，Zhou 等学者（2020）在 *Information Processing & Management* 期刊中分析了算法推荐机制对内容创作行为的影响。研究表明，基于推荐算法的内容分发模式虽然能够提升内容曝光率和用户活跃度，但也可能引导创作者过度关注外部反馈，从而削弱其对创作节奏和内容体系的自主规划能力。该研究指出，在以个人创作为核心的平台设计中，若过度强调流量导向，可能不利于创作者进行长期、系统性的内容积累。  此外，一些研究从系统设计和用户体验角度出发，探讨了数字内容平台在功能复杂度与使用门槛之间的平衡问题。相关研究认为，功能结构过于复杂或商业属性较强的平台，往往会增加用户使用负担，不利于普通用户进行长期内容记录和管理；相反，以个人为中心、功能结构清晰、操作流程简洁的平台，更有助于支持个人创作记录和内容沉淀。这类研究为本课题在平台设计中强调“轻量化创作记录与管理”、弱化流量干扰、强化内容整理与回顾功能提供了重要参考。  **（3）**    **现状总结与不足**  综合国内外相关研究可以看出，现有研究在个人知识管理理论、数字内容平台机制以及内容管理系统工程实现等方面已取得一定成果，普遍肯定了记录结构化、内容沉淀和用户体验对个人创作与知识积累的重要作用。然而，从整体来看，现有研究和平台应用仍存在一定不足：一是多数平台以成果展示或内容传播为核心，缺乏对个人创作过程的系统化、长期记录支持；二是部分系统功能复杂、商业属性较强，不利于普通创作者进行轻量化创作管理；三是针对“以个人创作记录为核心、兼顾作品管理与适度分享”的综合性平台研究相对较少。因此，有必要设计并实现一套轻量化、以创作过程记录为核心的个人创作记录与作品分享平台，以弥补现有研究和应用中的不足，这正是本课题的主要研究切入点。 | 3.  课题设计（或研究）的内容   本课题以个人创作记录与作品分享平台为研究对象，结合当前 Web 应用开发中常用的前后端分离技术体系，对系统的整体架构、功能模块及关键实现方案进行设计与实现。课题通过系统化的需求分析与工程化实现过程，完成从平台功能设计到系统部署的全过程研究，最终构建一个支持创作过程记录、作品集中管理与展示的综合性 Web 平台。  **（****1****）核心研究目标**  ①设计并实现以个人创作记录为核心的作品管理与分享平台：构建一套基于 Spring Boot + Vue 的个人创作记录与作品分享平台，实现个人创作内容的集中化、结构化管理，支持创作记录的长期保存、整理与回顾，避免创作内容分散存储和难以管理的问题。  ②构建轻量化、可扩展的前后端分离系统架构: 采用前后端分离的  B/S 架构，通过 RESTful 接口实现前后端数据交互，明确系统各层职责，提升系统的可维护性和扩展性，确保平台能够在毕业设计周期内稳定运行，并具备后续功能扩展的可能性。  ③提升个人创作管理与交流体验: 在满足创作记录与管理需求的基础上，引入适度的作品分享与互动功能，使创作者能够在低干扰环境中展示作品、获取反馈，从而增强持续创作动力和平台使用价值。   **（****2****）主要研究内容**  ①需求分析与系统功能规划模块：结合个人创作者的实际使用场景，对系统用户进行角色划分，包括普通用户、创作者和管理员。分析各类用户在创作记录、作品管理、作品展示和系统维护等方面的功能需求，明确系统的功能边界和业务流程，为后续系统设计提供依据。  ②系统总体架构设计模块：系统采用基于 B/S 架构的前后端分离模式。后端以 Spring Boot 框架为核心，负责业务逻辑处理和数据管理；前端采用 Vue 3 框架，结合  Element Plus 实现页面展示与用户交互。通过 RESTful API 实现前后端通信，构建结构清晰、职责明确的系统架构。  ③用户管理与权限控制模块：实现用户注册、登录及基本信息管理功能，引入基于 JWT 的身份认证机制，对不同角色用户的访问权限进行控制，保障系统数据安全性和操作规范性，为平台稳定运行提供基础支撑。  ④创作记录与作品管理模块：作为系统的核心功能模块，实现创作内容的新增、编辑、保存与删除等操作，支持对创作内容进行分类、标签管理和创作说明记录，并按照时间顺序对创作内容进行管理，方便用户对个人创作过程进行长期整理与回顾。  ⑤后台管理模块：面向系统管理员，实现对用户信息、作品内容和互动数据的管理与维护功能，保障平台内容的规范性和系统的正常运行。  ⑥数据库设计与数据管理模块：基于 MySQL 关系型数据库，对系统核心数据进行建模设计，包括用户信息表、作品信息表、评论数据表等，合理设置数据表结构和索引，提高数据存储的规范性和查询效率。  ⑦系统测试与优化模块：在系统开发完成后，对平台进行功能测试和基础性能测试，验证系统功能的正确性和运行稳定性，并根据测试结果对系统功能和交互体验进行优化，确保系统满足毕业设计任务书要求。  （3）拟解决的关键问题  ①个人创作内容的结构化建模与长期管理问题：个人创作内容具有类型多样、结构松散和更新频繁等特点，如何在保证系统轻量化的前提下，对创作记录、作品说明、标签分类以及时间信息进行统一建模，是本课题需要解决的关键问题之一。  ②前后端分离架构下的用户身份认证与权限控制问题：在基于 Spring Boot + Vue 的前后端分离系统中，如何实现安全、可靠的用户身份认证与权限控制，是系统设计中的技术难点。  ③创作记录管理与作品展示功能的协同实现问题：平台既需要支持个人创作过程的持续记录，又要兼顾作品的展示与互动功能。如何在功能设计上平衡“记录”和“展示”之间的关系，避免系统过度偏向内容发布而忽视创作过程管理，是本课题需要重点考虑的问题。 |                                                              |          |                |          |          |
| 4设计（或研究）方法   本课题属于工程设计类研究，采用软件工程方法体系组织研究与开发过程，整体遵循“文献与现状分析—需求分析—系统设计—模块实现—测试验证—优化完善”的技术路线。为保证研究与实现的科学性和可落地性，本课题具体主要采用以下研究方法：  （1）文献研究法：通过查阅个人知识管理、内容创作平台、前后端分离架构以及相关工程实现案例等文献资料，明确课题研究背景、研究现状和技术发展趋势，为系统功能设计和技术选型提供理论依据。  （2）需求调研与分析法：围绕个人创作者“记录—整理—展示—互动反馈”的使用场景，对普通用户、创作者与管理员三类角色的需求进行分析，明确系统功能边界、业务流程与非功能需求，形成需求规格说明，为系统设计提供输入。  （3）模块化设计法：在总体架构确定后，采用模块化设计思想对系统进行功能分解，将系统划分为用户与权限、创作记录与作品管理、作品展示与互动、后台管理、数据管理等模块。通过明确模块职责与接口边界，降低系统复杂度，提高系统开发效率与后期维护扩展能力。  （4）迭代开发法：系统开发过程中采用迭代增量方式推进：优先实现核心业务闭环（注册登录—创作记录—作品展示），再逐步补充互动、后台管理、统计与优化等功能，通过多轮“实现—验证—改进”持续提高系统完整性与可用性。  （5）测试验证法：在开发各阶段结合测试验证方法开展功能测试与基础性能测试，对关键业务流程、异常场景及权限控制进行验证；根据测试结果修复缺陷并优化数据库查询与接口响应，以保证系统稳定运行并满足毕业设计要求。 |                                                              |          |                |          |          |
